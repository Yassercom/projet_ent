Ce document détaille la création et l'implémentation d'une page de connexion moderne et professionnelle pour l'Environnement Numérique de Travail (ENT) de l'École Supérieure de Technologie de Salé, faisant partie de l'Université Mohammed V de Rabat.

5. Le Composant LoginPage

5.1. Structure TypeScript

Le composant utilise TypeScript pour définir les types de données :

// Types pour les rôles utilisateur
type UserRole = 'student' | 'professor' | 'admin';

// Interface pour l'état du formulaire
interface FormState {
  role: UserRole;
  identifier: string;
  password: string;
}

// Interface pour les erreurs du formulaire
interface FormErrors {
  identifier?: string;
  password?: string;
}

5.2. État du Composant

Utilisation des hooks React pour gérer l'état :

const [formState, setFormState] = useState({
  role: 'student',
  identifier: '',
  password: '',
});

const [formErrors, setFormErrors] = useState({});
const [isSubmitting, setIsSubmitting] = useState(false);

5.3. Fonctions de Validation

La fonction validateForm vérifie que les champs sont remplis correctement :

const validateForm = (): boolean => {
  const errors: FormErrors = {};
  
  if (!formState.identifier.trim()) {
    errors.identifier = 'L\'identifiant est requis';
  } else if (formState.role !== 'student' && !formState.identifier.endsWith('@um5.ac.ma')) {
    errors.identifier = 'Veuillez saisir une adresse email valide se terminant par @um5.ac.ma';
  }
  
  if (!formState.password) {
    errors.password = 'Le mot de passe est requis';
  }
  
  setFormErrors(errors);
  return Object.keys(errors).length === 0;
};
Point important : Pour les enseignants et administrateurs, nous vérifions que l'email se termine par "@um5.ac.ma".

5.4. Gestion des Événements

La fonction handleInputChange met à jour l'état du formulaire :

const handleInputChange = (e: React.ChangeEvent) => {
  const { name, value } = e.target;
  setFormState(prev => ({ ...prev, [name]: value }));
  
  // Effacer l'erreur quand l'utilisateur commence à taper
  if (formErrors[name as keyof FormErrors]) {
    setFormErrors(prev => ({ ...prev, [name]: undefined }));
  }
};

La fonction handleSubmit gère la soumission du formulaire :

const handleSubmit = (e: React.FormEvent) => {
  e.preventDefault();
  
  if (validateForm()) {
    setIsSubmitting(true);
    
    // Simulation d'appel API
    setTimeout(() => {
      console.log('Credentials submitted:', formState);
      setIsSubmitting(false);
    }, 1500);
  }
};

10. Concepts Clés Expliqués

10.1. React Hooks

Ce projet utilise useState pour gérer l'état du composant :

  • useState: Permet de gérer l'état local d'un composant

10.2. TypeScript

TypeScript nous permet de définir des types pour nos données :

  • type UserRole = 'student' | 'professor' | 'admin': Un type union qui limite les valeurs possibles
  • interface FormState {...}: Une interface qui définit la structure de l'état du formulaire

10.3. Tailwind CSS

Nous utilisons des classes utilitaires pour styler les éléments :

  • flex flex-col md:flex-row: Disposition flexible qui change selon la taille de l'écran
  • text-primary: Couleur personnalisée définie dans tailwind.config.js
  • rounded-xl: Coins arrondis
  • transition-all duration-200: Animations fluides

10.4. Validation Conditionnelle

La validation change selon le rôle de l'utilisateur :

  • Pour les étudiants : Vérification que le champ n'est pas vide
  • Pour les enseignants et administrateurs : Vérification que l'email se termine par "@um5.ac.ma"

11. Conclusion

Ce projet démontre comment créer une page de connexion moderne et professionnelle avec React, TypeScript et Tailwind CSS. Il est conçu pour être facilement compréhensible et modifiable, tout en respectant les meilleures pratiques de développement.

Pour aller plus loin : Vous pourriez ajouter une intégration back-end réelle, améliorer l'accessibilité, ou ajouter des fonctionnalités comme la persistance du rôle choisi.
